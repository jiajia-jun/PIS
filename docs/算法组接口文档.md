# PIS-web 算法组接口文档（拼接脚本 `stitch.py`）
## 注：{任务目录}后端会提供，算法组无需操心
## 1. 定位

| 脚本 | 路径 | 参数个数 | 执行时机 |
|---|---|---|---|
| **stitch.py**（本文档） | `./scripts/stitch.py` | **1 个** | 先执行 |
| eval_pipeline.py（见分析组文档） | `./scripts/eval_pipeline.py` | 2 个 | 拼接成功后执行 |

---

## 2. 调用方式

```bash
{python解释器} ./scripts/stitch.py {任务目录}
```

示例：
```bash
python3 ./scripts/stitch.py ./store/uploads/abc-123-def/
```

- **只收 1 个参数**：任务目录路径
- 超时限制：**普通模式 30 秒，超能模式 300 秒**

---

## 3. 目录结构

Go 后端在调用脚本前已经创建好以下目录结构：

```
{任务目录}/                         # ← CLI 第1个参数
├── input/                          # ← 【读】Go 已创建，图片已写入，只读
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── ...
└── result/                         # ← 【写】Go 已创建空目录，等脚本产出
```

---

## 4. 读：从 `{任务目录}/input/` 读取原始图片
- 🚨要实现的就是对`input/`目录的读取

该目录由 Go 后端在上传时创建，图片也已写入。Python 被调用时目录已就绪，**只读不写**。

| 项目 | 说明 |
|---|---|
| 格式 | JPG/PNG |
| 数量 | 2–20 张（超限自动截断前 20 张，不足 2 张报错） |
| 命名 | 保持原始文件名，按文件名升序排序 |
| 尺寸 | 不保证一致（脚本自动缩放到最长边 2000px） |
| 排序 | 按文件名排序 = 用户上传顺序 |

---

## 5. 写：产出到 `{任务目录}/result/`
- 🚨要实现的就是对`result/`目录的读取

该目录由 Go 后端在上传时创建（空目录），脚本**无需创建**，直接写入即可。

必须产出 **3 种文件**：

### 5.1 `result.jpg` — 拼接后的全景图

| 项目 | 说明 |
|---|---|
| 格式 | JPEG |
| 质量 | 建议 85–95 |

### 5.2 中间数据文件（供分析组消费）

拼接成功后额外产出 3 个文件：

| 文件 | 格式 | 说明 |
|---|---|---|
| `result_info.json` | JSON | 逐对匹配统计 `{pairs: [{total_matches, inlier_num}, ...], total_time}` |
| `H_list.npy` | NumPy | 相邻图对单应矩阵 `(n,3,3) float64` |
| `inliers_list.pkl` | Pickle | 相邻图对内点坐标 `[Nx4 ndarray, ...]`，每行 `[x1,y1,x2,y2]` |

> 这些文件仅供 eval_pipeline.py 读取，前端不直接消费。

### 5.3 `meta.json` — 执行结果元信息（算法组一定要提供这个数据）

```json
{
    "status": "ok",
    "keypoints": 150,
    "cost_ms": 3200,
    "error": ""
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `status` | string | ✅ | `"ok"` 成功 / `"error"` 失败 |
| `keypoints` | int | ✅ | 特征点匹配数（失败填 0） |
| `cost_ms` | int64 | ✅ | 脚本自身执行耗时（毫秒，失败填 0） |
| `error` | string | ✅ | 成功为空串，失败写原因 |

---

## 6. 错误处理

> ⚠️ **失败时也必须写 `meta.json`**，否则 Go 后端无法解析任务结果。

失败示例：
```json
{
    "status": "error",
    "keypoints": 0,
    "cost_ms": 0,
    "error": "重叠区域不足，无法拼接"
}
```

失败时**不生成** `result.jpg`。

---

## 7. 完整流程

```
Go 后端                                  stitch.py
───────                                  ─────────
1. 生成 task_id (UUID)
2. 创建 {任务目录}/input/  保存图片
3. 创建 {任务目录}/result/（空目录）
4. 执行 python3 stitch.py {任务目录} ──→ 5. 读 input/ 中的原图（≥2 张，≤20 张）
                                          6. SIFT 特征缓存 → 逐对匹配 → RANSAC
                                          7. OpenCV Stitcher 拼接（关闭波浪校正）
                                          8. 写 result/result.jpg
                                          9. 写 result/meta.json
                                          10. 写 result_info.json + H_list.npy + inliers_list.pkl
11. 读 result/meta.json              ←──
12. 更新数据库
13. 拼接成功后，继续调用 eval_pipeline.py
```

---

## 8. 速查总表

| | 说明                                              |
|---|-------------------------------------------------|
| **参数** | 1 个：任务目录路径                                      |
| **读什么** | `input/` （Go 后端会创建好，直接读取）下的 JPG 原图              |
| **写什么** | `result.jpg` + `meta.json`                      |
| **写到哪里** | `result/`（Go 后端创建，直接写入就好了）                      |
| **失败行为** | 必须写 `meta.json`（status="error"），不写 `result.jpg` |
| **超时** | 普通 30 秒 / 超能 300 秒                                            |
