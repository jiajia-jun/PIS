# PIS-web 算法组接口文档

## 1. 概述

本文档定义 Go 后端与 Python 拼接算法之间的**接口规范**。Go 后端通过命令行调用 Python 脚本，双方通过**文件系统 + JSON** 进行数据交换。

## 2. 调用方式

### 2.1 命令行格式

```
{python解释器路径} {脚本路径} {任务目录路径}
```

示例：
```bash
python3 /path/to/stitch.py /path/to/store/uploads/abc-123-def/
```

- `python解释器路径` 和 `脚本路径` 由后端配置决定，**Python 侧无需关心**
- **最后一个参数始终是任务目录路径**，Python 脚本只接收这一个参数

### 2.2 超时限制

Go 后端对 Python 调用设置了 **30 秒超时**。超过 30 秒进程会被强制终止，任务标记为失败。

---

## 3. 输入规范

### 3.1 任务目录结构

Go 后端在调用 Python 前会准备好以下目录：

```
./store/uploads/{task_id}/
├── input/                    # 用户上传的原始图片
│   ├── img_001.jpg
│   ├── img_002.jpg
│   ├── img_003.jpg
│   └── ...
└── result/                   # 空目录，等待 Python核心功能文件 产出
```
**🚨🚨后端会把用户输入存在一个名为`input/`的文件夹里面，Python组设计的时候记得注意一下读取这个文件夹下的数据**
### 3.2 输入说明

| 项目 | 说明 |
|---|---|
| 图片格式 | JPG |
| 图片数量 | 不固定 |
| 图片顺序 | 按文件名字典序排列，对应用户上传顺序 |
| 图片尺寸 | 不保证一致，算法需支持不同分辨率输入 |

## 🚨4. 输出规范（这里需要算法组注意一下）
- 这里仅作为一个暂时的demo来设计（后续再进行调整），数据分析组可以继续，算法组试试可不可以实现一下这个功能，
因为这些数据我打算放在历史记录里面的
- 至于数据分析的话就按照老师给的图片的那几个指标来

Python 脚本执行完毕后，**必须在 `result/` 目录下生成以下文件**：
```
{task_id}/
└── result/
    ├── meta.json             # 【必须】执行结果信息
    └── result.jpg            # 【必须】拼接后的全景图
```

### 4.1 meta.json 格式（执行结果的信息）

```json
{
    "status": "ok",         
    "keypoints": 150,
    "cost_ms": 3200,
    "error": ""
}
```

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `status` | string | ✅ | `"ok"` 表示成功，`"error"` 表示失败 |
| `keypoints` | int | ✅ | 特征点匹配总数 |
| `cost_ms` | int64 | ✅ | 算法执行耗时，单位**毫秒** |
| `error` | string | ✅ | 错误信息。成功时为空字符串 `""`，失败时填写具体错误原因 |

### 4.2 result.jpg 说明

| 项目 | 说明 |
|---|---|
| 格式 | JPEG |
| 内容 | 拼接完成的全景图 |
| 质量 | 建议 85-95（平衡文件大小与画质） |

**失败时不生成 result.jpg 即可**，Go 后端只读取 success 状态的任务结果图，对 error 状态的任务进行错误处理。

---

## 🚨5. 错误处理

当算法无法完成拼接时，需在 `meta.json` 中返回错误信息：

```json
{
    "status": "error",
    "keypoints": 0,
    "cost_ms": 0,
    "error": "输入图像重叠区域不足，无法拼接"
}
```

- `status` 设为 `"error"`
- `error` 字段填写**可读**的文本错误描述（支持直接写中文） Go 后端会将此错误信息存入数据库并展示给用户

## 6. 完整流程示意

```
Go 后端区                                         Python 核心功能区
───────                                           ──────────
1. 用户上传图片
2. 存入 input/ 目录
3. 运行py核心功能文件                        ──→ 4. 读取 input/ 目录中的图片
                                                5. 执行拼接算法
                                                6. 生成 result/result.jpg
8. 读取 result/meta.json                    ←── 7. 写入 result/meta.json 
9. 更新数据库（状态、耗时、特征点数）
10. 注册 10 分钟后清理 input/ 文件
```

---

## 7. 环境要求

| 项目        | 说明 |
|-----------|---|
| Python 版本 | 需支持命令行直接调用 |
| 依赖库       | 由算法组自行管理（OpenCV 等） |
| 结果输出      | 必须有权在 `result/` 目录下创建文件 |

---

## 8. 调试建议（这段话 Python组 不管也没事）

- 后端启动时默认使用 **Mock 引擎**（不调用 Python），开发阶段需将 `config.yaml` 中 `engine.type` 改为 `python` 才会真实调用脚本
- 任务目录下的文件不会被立即删除，**保留 10 分钟**，可利用这段时间手动检查 input/output 文件
- 请求日志和任务日志分别记录在 `logs/access.log` 和 `logs/task.log`
