# PIS-web 三组职责

> 面向答辩 PPT，精炼版。一页一张幻灯片。

---

## 后端组

| | |
|---|---|
| **技术栈** | Go + Gin + GORM + MySQL |
| **核心职责** | Web 服务 + 异步调度 + 数据管理 |

**三件事：**

1. **请求接入** — 9 个 REST API，文件上传（200MB 限制 + 路径穿越检查）、任务查询、历史分页、图片/分析文件服务
2. **调度系统** — 20 协程 Worker Pool + 有缓冲 Channel，双模式超时（30s/300s），满负载背压保护，三级容错（拼接/分析/缩略图失败互不影响）
3. **数据管理** — Task 状态机（pending→processing→completed/failed），40 条历史自动淘汰（删 DB + 删磁盘），双日志分离

| 输入 | 输出 |
|---|---|
| 前端 HTTP 请求 | REST API JSON |

---

## 算法组

| | |
|---|---|
| **技术栈** | Python + OpenCV（stitch.py 434行） |
| **核心职责** | 全景拼接管线 |

**完整管线：**

```
原始图片 → 预处理(排序/缩放2000px/限20张) → SIFT特征提取(回退ORB)
→ BFMatcher+KNN+Lowe's Ratio(0.7) → RANSAC单应矩阵
→ OpenCV Stitcher(球面/平面双模式) → 后处理(限4000px/黑边裁剪)
→ 输出
```

**6 个产出文件：** result.jpg（拼接结果）、meta.json（status/keypoints/cost_ms/error）、H_list.npy + inliers_list.pkl + result_info.json（供分析组消费）

| 输入 | 输出 |
|---|---|
| `store/uploads/{id}/input/` | `store/uploads/{id}/result/` |

---

## 分析组

| | |
|---|---|
| **技术栈** | Python（eval_core.py 2270行 + eval_pipeline.py 526行） |
| **核心职责** | 拼接质量量化评估 |

**适配器 + 引擎架构：**

- **eval_core.py** — 纯计算引擎，读 SIFT/RANSAC 数据算六项指标，生成图表。不依赖 PIS-web，可独立运行。
- **eval_pipeline.py** — 管线适配器，0 行算法代码。入参桥接(stitch格式→引擎格式) → 调引擎 → 出参转换(引擎格式→前端格式)

**六项指标：** 内点率、重投影 RMSE、全景 SSIM、画布占比、清晰度保持率、综合得分

| 输入 | 输出 |
|---|---|
| stitch.py 中间数据 + 结果图 | 10 张 PNG + 2 个 JSON |

---

## 三组通信

```
前端 ──HTTP──▶ 后端(Go) ──os/exec──▶ 算法(stitch.py)
                    │                      │
                    │       result.jpg + meta.json + 中间数据
                    │                      │
                    └──os/exec──▶ 分析(pipeline → core)
                                           │
                                  10 PNG + 2 JSON ──▶ 前端
```

全部基于**文件系统 + 约定目录**，零网络协议，零编码开销。独立开发，靠文件名和 JSON 字段约定对齐。
