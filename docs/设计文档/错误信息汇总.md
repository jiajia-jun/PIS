# PIS-web 错误信息汇总

本文档汇总系统中所有可能产生的错误信息，按层级和流向排列，方便排查和定位。

---

## 1. 启动阶段（Fatal 级，进程退出）

| 错误信息 | 来源 | 触发条件 |
|---|---|---|
| `加载配置失败: {err}` | `internal/app/app.go` | `config.yaml` 不存在或格式错误 |
| `环境变量 DATABASE_PIS 未设置` | `internal/app/app.go` | 未设置 MySQL 连接串环境变量 |
| `数据库连接失败: {err}` | `internal/database/db.go` | MySQL 不可达 / 密码错误 / 库不存在 |
| `自动建表失败: {err}` | `internal/database/db.go` | GORM AutoMigrate 异常 |
| `数据库初始化失败: {err}` | `internal/app/app.go` | 上述 DB 错误的透传 |
| `服务启动失败: {err}` | `internal/app/app.go` | 端口被占用 |

## 2. API 层（返回前端）

### 2.1 `POST /api/upload`

| code | message | 触发条件 |
|---|---|---|
| 413 | `文件总大小超过{200}MB限制` | Content-Length 超过配置值 |
| 413 | `文件总大小超过限制` | `http.MaxBytesReader` 触发（流式上传超限） |
| 400 | `文件解析错误` | multipart 表单解析失败 |
| 400 | `未选择图片` | 表单中 `images` 字段为空或无文件 |
| 500 | `创建任务失败` | 数据库写入 Task 记录失败 |
| 500 | `创建目录失败` | 磁盘满 / 权限不足 |
| 500 | `保存文件失败` | 磁盘满 / 写入异常 |
| 503 | `系统繁忙，请稍后重试` | 任务队列满，3 秒内未提交成功 |

### 2.2 `GET /api/task/:task_id`

| code | message | 触发条件 |
|---|---|---|
| 400 | `缺少 task_id` | URL 参数为空 |
| 404 | `任务不存在` | task_id 在数据库中不存在 |

> 完成/失败时 `data` 内附带字段 `error`（仅 failed 状态），内容透传自 Python `meta.json` 的 `error` 字段。

### 2.3 `GET /api/result/:task_id`

| code | message | 触发条件 |
|---|---|---|
| 400 | `缺少 task_id` | URL 参数为空 |
| 404 | `任务不存在` | task_id 在数据库中不存在 |
| 404 | `结果尚未生成` | 任务状态不是 completed |
| 404 | `结果文件不存在` | DB 记录 completed 但磁盘文件缺失 |

### 2.4 `GET /api/thumbnail/:task_id`

| code | message | 触发条件 |
|---|---|---|
| 404 | `结果尚未就绪` | 任务不存在或未完成 |

### 2.5 `GET /api/analysis/:task_id/:filename`

| code | message | 触发条件 |
|---|---|---|
| 400 | `缺少参数` | task_id 或 filename 为空 |
| 404 | `分析文件不存在` | 图表 PNG 或 eval_result.json 不存在 |

### 2.6 `GET /api/input/:task_id/:filename`

| code | message | 触发条件 |
|---|---|---|
| 400 | `缺少参数` | task_id 或 filename 为空 |
| 400 | `非法文件名` | 文件名含 `..` 等路径穿越字符 |
| 404 | `任务不存在` | task_id 在数据库中不存在 |
| 404 | `文件不存在` | 磁盘上原始图片不存在 |

### 2.7 `GET /api/history`

| code | message | 触发条件 |
|---|---|---|
| 500 | `查询失败` | 数据库查询异常 |

## 3. 引擎 / Worker 层（写入 DB `error_msg`，前端可见）

### 3.1 通用 Worker 错误

| 错误信息 | 来源 | 触发条件 |
|---|---|---|
| `任务队列已满，请稍后重试` | `internal/worker/pool.go` | 上传时 Channel 满 → 返回 503 |

### 3.2 Mock 拼接引擎

| 错误信息 | 触发条件 |
|---|---|
| `读取输入目录失败: {err}` | `input/` 目录不可读 |
| `输入目录无图片文件` | input/ 目录存在但无图片 |
| `复制结果图失败: {err}` | 磁盘写入失败 |

### 3.3 Python 拼接引擎 (`script` → Go 透传)

| 错误信息 | 触发条件 |
|---|---|
| `Python 执行超时: context deadline exceeded` | Python 进程超过 30 秒未结束 |
| `Python 执行失败: {err}\n输出: {stderr}` | Python 进程非零退出（非超时） |
| `读取 meta.json 失败: {err}` | `result/meta.json` 不存在或格式错误 |
| `解析 meta.json 失败: {err}` | meta.json 存在但 JSON 解析失败 |
| `拼接失败（未知原因）` | meta.status="error" 但 error 字段为空 |

### 3.4 Python 分析引擎

| 错误信息 | 触发条件 |
|---|---|
| `创建分析目录失败: {err}` | 磁盘满 / 权限不足 |
| `分析超时: {err}` | Python 进程超过 30 秒 |
| `分析失败: {err}\n输出: {stderr}` | Python 非零退出 |

> 分析失败仅记 `logs/task.log` warning，**不影响**拼接结果（任务仍标记 completed）。

### 3.5 C++ 引擎

| 错误信息 | 触发条件 |
|---|---|
| `c++引擎待开发` | 配置了 cpp 引擎但未实现 |

### 3.6 缩略图生成

| 错误信息 | 触发条件 |
|---|---|
| `打开原图失败: {err}` | `result.jpg` 无法打开 |
| `创建缩略图失败: {err}` | 磁盘写入失败 |

> 缩略图失败仅记日志 warning，不影响拼接结果。

### 3.7 Service / DB

| 错误信息 | 触发条件 |
|---|---|
| `创建任务失败: {err}` | 数据库插入 Task 记录失败 |

## 4. Python stitch.py（写入 `meta.json` 的 `error` 字段）

以下错误由算法脚本产生，通过 `meta.json` → Go `HandleResult` → DB `error_msg` → 前端展示。

### 4.1 目录/文件错误

| error | 触发条件 |
|---|---|
| `任务目录不存在: {path}` | `sys.argv[1]` 路径不存在 |
| `输入目录不存在: {path}` | `{任务目录}/input/` 不存在 |
| `输入目录中未找到图片文件: {path}` | input/ 下无 JPG/PNG 文件 |

### 4.2 图片质量错误

| error | 触发条件 |
|---|---|
| `{filename}: 无法解码，文件可能已损坏` → 累积后: `所有图片均无法读取: {details}` | 所有图片均无法被 cv2 解码 |
| `{filename}: 有效像素过少，疑似纯色/空白图` → 累积后可能触发数量不足 | 图片内容像素太少 |
| `有效图片不足（仅 N 张），至少需要 2 张图片才能拼接` | 有效图片 < 2 张 |

### 4.3 拼接算法错误（OpenCV）

| error | 触发条件 |
|---|---|
| `需要更多图像（图片间重叠区域不足或特征点太少）` | `Stitcher_ERR_NEED_MORE_IMGS` |
| `单应矩阵估计失败（图片间匹配关系不稳定）` | `Stitcher_ERR_HOMOGRAPHY_EST_FAIL` |
| `相机参数调整失败（图片视角差异过大或排列不合理）` | `Stitcher_ERR_CAMERA_PARAMS_ADJUST_FAIL` |
| `OpenCV 拼接失败 (错误码 {code})` | 其他未分类 OpenCV 错误码 |

### 4.4 写入错误

| error | 触发条件 |
|---|---|
| `写入拼接结果失败: {path}` | `cv2.imwrite` 返回 False（磁盘满/权限） |
| `严重: 写入 meta.json 失败: {err}` → **stderr** | meta.json 写入失败（最坏情况，前端看不到此信息） |

## 5. 错误流转路径

```
                        ┌──────────────────┐
                        │   stitch.py      │
                        │  meta.json        │
                        │  error: "xxx"     │
                        └───────┬──────────┘
                                │ PythonEngine.Run()
                                ▼
                    ┌───────────────────────┐
                    │ worker/pool.go        │
                    │ meta.Status=="error"? │
                    │ → skip analysis       │
                    └───────┬───────────────┘
                            │ HandleResult()
                            ▼
                    ┌───────────────────────┐
                    │ service/task_service  │
                    │ meta.Status=="error"? │
                    │ → task.ErrorMsg =     │
                    │   meta.Error          │
                    │ → status = failed     │
                    └───────┬───────────────┘
                            │ DB: error_msg 字段
                            ▼
                    ┌───────────────────────┐
                    │ GET /api/task/:id     │
                    │ resp.data.error       │
                    └───────┬───────────────┘
                            │ 前端轮询
                            ▼
                    ┌───────────────────────┐
                    │  TaskDetailView.vue   │
                    │  el-result            │
                    │  sub-title="error"    │
                    └───────────────────────┘
```
