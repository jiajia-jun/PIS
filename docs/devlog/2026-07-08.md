### 6. Windows glob 文件重复加载修复
**状态**: 未提交

Windows 上 `glob` 大小写不敏感，`*.jpg` 和 `*.JPG` 匹配同一文件，每张图片被加载两次，导致不相关图片也被"拼接成功"。修复：`os.path.normcase` + `set` 去重。

### 10. stitch.py 五项优化
**提交**: `598f322`

| 优化 | 说明 |
|---|---|
| 描述子缓存 | `_detect_all` 一次性提取所有图片 SIFT 特征，逐对匹配复用 kp/des；N 张图从 O(2N) 次描述子提取降为 O(N) |
| 删除 `count_keypoints` | 特征点数直接从 `_detect_all` 缓存取 `len(kp)`，消除第三轮 SIFT 提取 |
| `MAX_IMAGES=20` 上限 | 手动调用 Python 脚本时防止内存溢出，超限打印警告并截断（与 Go 后端对齐） |
| 统一耗时字段 | `cost_ms` 在 `main()` 中计算一次，同时写入 `meta.json.cost_ms` 和 `result_info.json.total_time`，消除两个字段取值偏差 |
| 磁盘空间检查 | `write_result` 中 `imwrite` 失败时用 `shutil.disk_usage` 区分为"磁盘满"还是"权限/路径问题"，提供具体 MB 数值 |
| 关闭 waveCorrection | `stitcher.setWaveCorrection(False)`，避免非水平排列图片序列产生波浪形畸变 |
