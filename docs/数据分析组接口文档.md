# PIS-web 数据分析组接口文档（分析脚本 `analysis.py`）

## 1. 定位

| 脚本 | 路径 | 参数个数 | 执行时机 |
|---|---|---|---|
| stitch.py（见算法组文档） | `./scripts/stitch.py` | 1 个 | 先执行 |
| **analysis.py**（本文档） | `./scripts/analysis.py` | **2 个** | 拼接成功后执行 |

> 拼接失败的任务**不会**调用分析脚本。

---

## 2. 调用方式

```bash
{python解释器} ./scripts/analysis.py {任务目录} {分析输出目录}
```

示例：
```bash
python3 ./scripts/analysis.py ./store/uploads/abc-123-def/ ./store/analysis/abc-123-def/
```

| 参数 | 含义 | 说明 |
|---|---|---|
| 第1个 | 任务目录 | 从此目录读取拼接结果 |
| 第2个 | 分析输出目录 | 往里面写 PNG 图表 |

- **收 2 个参数**
- 超时限制：**30 秒**
- 分析失败**不影响拼接结果**，拼接仍标记为 completed

---

## 3. 目录结构

脚本被调用时，任务目录结构如下（拼接已完成）：

```
{任务目录}/                         # ← CLI 第1个参数
├── input/                          # ← 【读】原始图片
│   ├── img_001.jpg
│   └── ...
└── result/                         # 拼接产出
    ├── meta.json                   # （Go 后端用，分析脚本无需读取）
    └── result.jpg                  # ← 【读】拼接全景图

{分析输出目录}/                      # ← CLI 第2个参数（不存在，脚本自行创建）
```

---

## 4. 读：从任务目录读取图片，独立生成分析数据

> 分析脚本**不读 meta.json**。meta.json 是 Go 后端用来给前端展示历史记录（状态、耗时、特征点数）的，与分析脚本无关。
>
> 分析脚本独立读取图片数据，自行计算分析指标并生成图表。
>
> 数据分析组请自行进行信息的读取

## 5. 写：产出 PNG 图表到 `{分析输出目录}`

### 5.1 `{分析输出目录}` 解释

`{分析输出目录}` 就是 CLI 的 **第2个参数**，Go 后端直接传过来的完整路径，例如：

```
./store/analysis/abc-123-def/
```

**脚本拿到手就能用，不需要拼路径。**

### 5.2 创建目录

> ⚠️ 该目录**不存在**，脚本需**自行创建**。

```python
import sys, os

out_dir = sys.argv[2]    # 例如: ./store/analysis/abc-123-def/
os.makedirs(out_dir, exist_ok=True)
```

就两行：拿参数 → 建目录。路径 Go 已经拼好了，直接用。

### 5.3 往里面写 PNG

建好目录后，图表直接往里写：

```python
import matplotlib.pyplot as plt

# 画图
plt.plot(...)
plt.savefig(os.path.join(out_dir, "cost_trend.png"))
```

### 5.4 产出规范

| 项目 | 说明 |
|---|---|
| 格式 | **PNG** |
| 数量 | 不固定，按分析维度决定 |
| 命名 | 语义化英文，不含中文。如 `cost_trend.png`、`keypoints_chart.png` |

最终产出示例：
```
./store/analysis/abc-123-def/      # ← 就是 sys.argv[2]，脚本自己建的
├── cost_trend.png
├── keypoints_chart.png
└── overview.png
```

### 5.5 建议分析维度（以老师要求为准）

- 拼接耗时统计
- 特征点数分布
- 成功率
- 图片数量与耗时关系

---

## 6. 前端如何访问

分析图表通过 API 暴露：

```
GET /api/analysis/{task_id}/{filename}
```

历史记录中每条已完成任务会携带 `analysis_urls`：

```json
{
    "task_id": "abc-123-def",
    "status": "completed",
    "analysis_urls": [
        "/api/analysis/abc-123-def/cost_trend.png",
        "/api/analysis/abc-123-def/keypoints_chart.png"
    ]
}
```

> 文件名需与脚本实际产出文件名一致，请与前后端沟通确定。

---

## 7. 容错

- 分析脚本执行失败 → 拼接依然标记为 completed，仅在后端日志 `logs/task.log` 中记录 warning
- 图表不存在时前端对应位置留空

---

## 8. 速查总表

| | 说明 |
|---|---|
| **参数** | 2 个：任务目录 + 分析输出目录 |
| **读什么** | `{任务目录}/input/` 原始图片 + `{任务目录}/result/result.jpg` 全景图 |
| **写什么** | PNG 图表（数量不固定） |
| **写到哪里** | `{分析输出目录}`（**目录不存在，脚本自己建**） |
| **失败行为** | 不影响拼接结果，后端仅记日志 |
| **超时** | 30 秒 |
