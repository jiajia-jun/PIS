# PIS-web 数据分析组接口文档（分析脚本 `analysis.py`）

## 1. 定位

| 脚本 | 路径 | 参数个数 | 执行时机 |
|---|---|---|---|
| stitch.py（见算法组文档） | `./scripts/stitch.py` | 1 个 | 先执行 |
| **analysis.py**（本文档） | `./scripts/analysis.py` | **2 个** | 拼接成功后执行 |

> 拼接失败的任务**不会**调用分析脚本。

---

## 2. 调用方式

```bash
{python解释器} ./scripts/analysis.py {任务目录} {分析输出目录}
```

示例：
```bash
python3 ./scripts/analysis.py ./store/uploads/abc-123-def/ ./store/analysis/abc-123-def/
```

| 参数 | 含义 | 说明 |
|---|---|---|
| 第1个 | 任务目录 | 从此目录读取拼接结果 |
| 第2个 | 分析输出目录 | 往里面写 PNG 图表 |

- **收 2 个参数**
- 超时限制：**30 秒**
- 分析失败**不影响拼接结果**，拼接仍标记为 completed

---

## 3. 目录结构

脚本被调用时，任务目录结构如下（拼接已完成）：

```
{任务目录}/                         # ← CLI 第1个参数
├── input/                          # ← 【读】原始图片
│   ├── img_001.jpg
│   └── ...
└── result/                         # 拼接产出（算法组）
    ├── meta.json                   # Go 后端用
    ├── result.jpg                  # ← 【读】拼接全景图
    ├── H_list.npy                  # ← 【读】单应矩阵列表（stitch.py 产出）
    ├── inliers_list.pkl            # ← 【读】内点坐标列表（stitch.py 产出）
    └── result_info.json            # ← 【读】匹配统计（stitch.py 产出）

{分析输出目录}/                      # ← CLI 第2个参数（Go 后端已创建，直接写入）
```

---

## 4. 读：从任务目录读取拼接结果与中间数据

分析脚本从 `{任务目录}/` 读取以下文件：

| 文件 | 路径 | 说明 |
|---|---|---|
| 原始图片 | `{任务目录}/input/` | JPG/PNG 原图，按文件名排序 |
| 拼接全景图 | `{任务目录}/result/result.jpg` | OpenCV 拼接结果 |
| 单应矩阵 | `{任务目录}/result/H_list.npy` | numpy stack 格式 (n,3,3) float64 |
| 内点坐标 | `{任务目录}/result/inliers_list.pkl` | pickle 格式，每个元素为 (N,4) 数组 |
| 匹配统计 | `{任务目录}/result/result_info.json` | `{pairs: [{total_matches, inlier_num}], total_time}` |

> 以上中间数据由 `stitch.py`（算法组）产出。分析脚本不读 `meta.json`。

## 🚨🚨🚨5. 写：产出 PNG 图表 + JSON 表格到 `{分析输出目录}`

### 5.1 `{分析输出目录}` 解释

`{分析输出目录}` 就是 CLI 的 **第2个参数**，Go 后端直接传过来的完整路径，例如：

```
./store/analysis/abc-123-def/
```

**Go 后端已经创建好该目录**，脚本拿到手就能直接往里写 PNG，**不需要自己建目录**。

### 5.2 往里面写 PNG

Go 后端在调用脚本前已经执行了 `os.MkdirAll`，所以脚本**不需要自己建目录**。

`sys.argv[2]` 拿到的就是一个已存在的目录路径，例如 `./store/analysis/abc-123-def/`，直接往里 `plt.savefig` 即可：
- 下面这段代码供数据组参考
```python
import sys, os
import matplotlib.pyplot as plt

out_dir = sys.argv[2]    # Go 已创建，例如 ./store/analysis/abc-123-def/

# 直接写入，无需 os.makedirs
plt.plot(...)
plt.savefig(os.path.join(out_dir, "cost_trend.png"))
```

> 对比：算法组的 `result/` 目录也是 Go 预先建好的，脚本直接用。两组逻辑完全一致——Go 管目录，脚本管内容。

### 5.3 产出规范

| 项目 | 说明 |
|---|------|
| 格式 | **PNG**（图表） + **JSON**（数据） |
| PNG 数量 | 固定 10 张 |
| JSON 数量 | 固定 2 个 |

最终产出示例：
```
./store/analysis/abc-123-def/      # ← 就是 sys.argv[2]，Go 后端已创建
├── eval_result.json               # 精简评估结果（前端摘要表格用）
├── full_metrics.json              # 全部分组指标（前端详细面板用）
├── metrics_dashboard.png          # 6 指标仪表盘总览
├── quality_radar.png              # 6 维综合雷达图
├── gauge_内点率.png               # 各指标独立大图 ×6
├── gauge_重投影RMSE.png
├── gauge_重叠区SSIM.png
├── gauge_总耗时.png
├── gauge_画布利用率.png
├── gauge_综合得分.png
└── metrics_table.png              # 指标明细数据表
```

### 5.4 `eval_result.json` 格式（精简摘要）

```json
{
  "用例名称": "abc-123-def",
  "状态": "成功",
  "内点率": 0.2498,
  "重投影RMSE": 0.9655,
  "重叠区SSIM": 0.0692,
  "有效画布占比": 0.9357,
  "清晰度保持率": 0.8129,
  "综合得分": 0.6131,
  "总耗时(秒)": 0.64,
  "警告": ""
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| 用例名称 | string | 任务目录名 |
| 状态 | string | "成功" 或 "失败" |
| 内点率 | number/null | RANSAC 内点数 / 总匹配对数 |
| 重投影RMSE | number/null | 重投影均方根误差（像素） |
| 重叠区SSIM | number/null | 重叠区域结构相似度 (0-1) |
| 有效画布占比 | number | 全景图中非黑像素占比 |
| 清晰度保持率 | number | 全景图/原图 Laplacian 方差比 |
| 综合得分 | number | 6 维归一化评分均值 |
| 总耗时(秒) | number | 拼接耗时 |
| 警告 | string | 计算过程中的警告信息 |

## 6. 前端如何访问

分析图表和表格通过 API 暴露：

```
GET /api/analysis/{task_id}/{filename}
```

历史记录中每条已完成任务会携带 `chart_urls` 和 `table_urls`：

```json
{
    "task_id": "abc-123-def",
    "status": "completed",
    "chart_urls": [
        "/api/analysis/abc-123-def/gauge_内点率.png",
        "/api/analysis/abc-123-def/gauge_重叠区SSIM.png",
        "/api/analysis/abc-123-def/gauge_综合得分.png",
        "/api/analysis/abc-123-def/gauge_总耗时.png",
        "/api/analysis/abc-123-def/gauge_画布利用率.png",
        "/api/analysis/abc-123-def/gauge_重投影RMSE.png",
        "/api/analysis/abc-123-def/metrics_dashboard.png",
        "/api/analysis/abc-123-def/metrics_table.png",
        "/api/analysis/abc-123-def/quality_radar.png"
    ],
    "table_urls": [
        "/api/analysis/abc-123-def/eval_result.json",
        "/api/analysis/abc-123-def/full_metrics.json"
    ]
}
```

前端任务详情页自动渲染：
- **评估摘要区**：6 指标表格（含优秀/一般/较差评级标签）+ 雷达图（可点击看大图）
- **详细数据面板**：折叠式 `el-tabs`，按 6 个分组展示 `full_metrics.json` 全部字段
- **其余图表**：`metrics_dashboard.png`、`metrics_table.png` 等以网格展示

> 文件名需与脚本实际产出文件名一致，请与前后端沟通确定。

---

## 7. 容错

- 分析脚本执行失败 → 拼接依然标记为 completed，仅在后端日志 `logs/task.log` 中记录 warning
- 图表不存在时前端对应位置留空

---

## 8. 速查总表

| | 说明                         |
|---|----------------------------|
| **参数** | 2 个：任务目录 + 分析输出目录          |
|**读什么**| 与算法组单独沟通，务必让算法组告知后端组数据传输流程 |
| **写什么** | PNG 图表（数量不固定）              |
| **写到哪里** | `{分析输出目录}`（Go 后端已创建，脚本直接写） |
| **失败行为** | 不影响拼接结果，后端仅记日志             |
| **超时** | 30 秒                       |
