# PIS-web 数据分析组接口文档（分析脚本）

## 1. 概述

本系统包含两个独立的 Python 脚本，由 Go 后端先后调度：

| 脚本 | 作用 | 执行时机 |
|---|---|---|
| **拼接脚本**（见《算法组接口文档》） | 图像拼接，产出全景图 + meta.json | 先执行 |
| **分析脚本**（本文档） | 生成分析图表 PNG | 拼接成功后执行 |

> 本文档仅描述分析脚本的接口规范。拼接脚本请参阅《算法组接口文档》。

---

## 2. 调用方式

### 2.1 命令行格式

```
{python解释器} {脚本路径} {任务目录}
```

示例：
```bash
python3 analysis.py E:\xxx\PIS-web\store\uploads\abc-123-def\
```

- python 解释器路径和脚本路径由后端 `config.yaml` 配置
- 最后一个参数是任务目录路径，脚本只收这一个参数

### 2.2 执行前置条件

分析脚本只在**拼接成功**后才被后端调用。拼接失败的或未完成的任务不会调用分析脚本。

### 2.3 超时限制

Go 后端对脚本调用设置了 **30 秒超时**。分析脚本执行失败不会影响拼接结果。

---

## 3. 输入规范

### 3.1 从哪里获取数据

分析脚本被调用时，任务目录结构如下：

```
{任务目录}/
├── input/                          # 用户上传的原图（可能仍在）
│   ├── img_001.jpg
│   └── ...
└── result/                         # ← 拼接脚本的产出，从这里读
    ├── meta.json                   # ← 拼接结果元信息
    └── result.jpg                  # ← 拼接后的全景图
```

### 3.2 可用数据来源

| 数据 | 路径 | 说明 |
|---|---|---|
| meta.json | `{任务目录}/result/meta.json` | 拼接结果：status、keypoints、cost_ms |
| result.jpg | `{任务目录}/result/result.jpg` | 拼接后的全景图 |
| 原图 | `{任务目录}/input/` | 用户上传的原始图片 |

---

## 4. 输出规范

### 4.1 写到哪里

> **输出路径：`store/analysis/{task_id}/`**

注意：task_id 即任务目录名（如 `abc-123-def`）。该目录需要分析脚本自行创建。

**完整路径示例**：
```
E:\xxx\PIS-web\store\analysis\abc-123-def\
```

### 4.2 产出文件

| 项目 | 说明 |
|---|---|
| 文件格式 | PNG |
| 文件数量 | 不固定，按老师要求的指标数量决定 |
| 命名规则 | 语义化，不含中文。如 `cost_trend.png`、`keypoints_chart.png` |

示例：
```
store/analysis/abc-123-def/
├── cost_trend.png
├── keypoints_chart.png
└── stitching_overview.png
```

### 4.3 分析维度

具体分析指标以后端老师给的图片为准，参考方向：

- 拼接耗时统计
- 特征点数分布
- 任务成功率统计
- 图片数量与耗时的关系

---

## 5. 前端如何获取图表

前端通过以下 API 获取图表文件：

```
GET /api/analysis/{task_id}/{filename}
```

示例：
```
GET /api/analysis/abc-123-def/cost_trend.png
```

在历史记录中，每个任务会携带 `analysis_urls` 字段：

```json
{
    "task_id": "abc-123-def",
    "status": "completed",
    "cost_ms": 3200,
    "keypoints": 150,
    "result_url": "/api/result/abc-123-def",
    "analysis_urls": [
        "/api/analysis/abc-123-def/cost_trend.png",
        "/api/analysis/abc-123-def/keypoints_chart.png"
    ],
    "created_at": "2026-07-06T14:30:22Z"
}
```

**注意**：`analysis_urls` 中的文件名需要与脚本实际产出的文件名完全一致。请与后端和前端沟通确定文件名列表。

---

## 6. 容错说明

- 分析脚本执行失败**不影响拼接结果**，拼接依然标记为 completed
- 分析失败会在后端日志 `logs/task.log` 中记录 warning
- 如果某些图表不存在，前端对应位置留空即可

---

## 7. 调试说明

- 后端默认使用 Mock 分析引擎（不调脚本）。开发时将 `config.yaml` 中 `engine.analysis_type` 改为 `python` 才会真实调用
- 拼接引擎和分析引擎可以独立切换 mock/python，联调方便
- 任务目录下的文件保留 10 分钟才清理
