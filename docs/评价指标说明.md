# 全景拼接评价指标说明

## 概述

每张全景拼接任务完成后，系统自动产出 **6 项评分** + **1 项综合得分**，全部归一化到 0~1 区间，1 为最优。

指标分两类：
- **过程指标**：基于中间数据（匹配信息、H 矩阵），反映拼接过程的可靠性
- **结果指标**：直接在最终全景图上计算，反映用户看到的实际画质

---

## 一、六项分项指标

### 1. 匹配质量（内点率）

| 项目 | 说明 |
|------|------|
| **含义** | 特征匹配的有效率：RANSAC 内点数 / 总匹配数 |
| **来源** | `stitch.py` 对相邻图对做 SIFT + BFMatcher + Lowe's ratio + RANSAC，汇总所有对 |
| **优秀** | \> 0.5 |
| **差** | < 0.2（大量误匹配，图片可能重叠不足或纹理重复） |

### 2. 对齐精度（重投影 RMSE）

| 项目 | 说明 |
|------|------|
| **含义** | 相邻图对的内点经 H 矩阵投影后，与目标点的平均像素误差 |
| **来源** | 所有相邻对的重投影误差取平均值 |
| **归一化** | `1.0 - RMSE / max(图像对角线 × 0.005, 1)` |
| **优秀** | \> 0.8（RMSE 远小于对角线 0.5%） |
| **差** | < 0.3（RMSE 接近对角线 0.5%，拼接错位明显） |

### 3. 拼接一致性（全景 SSIM）

| 项目 | 说明 |
|------|------|
| **含义** | 全景图自身的结构连续性：滑动窗口遍历有效区域，每窗口与自身偏移 1 像素后计算 SSIM，汇总均值 |
| **来源** | 滑动窗口（32×32，步长 24）遍历全景图有效区域，每窗口做水平/垂直各 1px 偏移 SSIM |
| **优秀** | \> 0.95 |
| **差** | < 0.8（拼接缝区域有明显结构断裂或鬼影） |

> **原理**：正常纹理偏移 1 像素后内容几乎不变 → SSIM ≈ 0.98；拼接缝处偏移 1 像素跨过缝线 → 结构突变 → SSIM 骤降。不依赖 H 矩阵，直接检验最终全景图的局部连续性。

### 4. 画布利用率（有效画布占比）

| 项目 | 说明 |
|------|------|
| **含义** | 全景图中非黑像素占比 |
| **来源** | 全景图灰度化后，统计亮度 \> 10 的像素比例 |
| **优秀** | \> 0.7 |
| **差** | < 0.4（大量黑边，图片排列或 warping 不合理） |

### 5. 清晰度保持率

| 项目 | 说明 |
|------|------|
| **含义** | 全景图清晰度 / 原图平均清晰度 |
| **来源** | Laplacian 方差：`Var(Laplacian(全景)) / Mean(Var(Laplacian(原图_i)))` |
| **归一化** | `min(1.0, retention)`，上限 1.0（拼接不应比原图更清晰） |
| **优秀** | \> 0.8 |
| **差** | < 0.4（拼接导致严重模糊，可能是 blending 过度平滑或分辨率损失） |

### 6. 运行效率

| 项目 | 说明 |
|------|------|
| **含义** | 拼接耗时评分 |
| **来源** | 总耗时（秒） |
| **归一化** | `1.0 - 总耗时 / (图片数量 × 10s)` |
| **优秀** | \> 0.8 |
| **差** | < 0.3（拼接耗时远超预期） |

---

## 二、综合得分

```
综合得分 = Mean(匹配质量, 对齐精度, 拼接一致性, 画布利用率, 清晰度保持率, 运行效率)
```

6 个维度的算术平均，权重相等。

---

## 三、辅助指标

| 字段 | 来源 | 含义 |
|------|------|------|
| `总匹配对数` | stitch.py | 所有相邻图对 knnMatch 匹配总数 |
| `RANSAC内点数` | stitch.py | 通过 Lowe's ratio + RANSAC 筛选的内点总数 |
| `keypoints` | stitch.py | 所有源图 SIFT 特征点总数（粗略复杂度指示） |
| `cost_ms` | stitch.py | 拼接总耗时（毫秒） |
| `全景SSIM_min` | test5.py | 全景切片 SSIM 最低值，指示最差缝线位置 |
| `全景SSIM_std` | test5.py | 全景切片 SSIM 标准差，指示拼接质量均匀性 |
| `重投影中位数` | test5.py | 重投影误差中位数 |
| `重投影P95` | test5.py | 重投影误差 95 分位数 |
| `空白占比` | test5.py | 1 - 有效画布占比 |

## 四、数据流

```
stitch.py
  ├─ meta.json          → Go Task 表（keypoints, cost_ms）
  ├─ result_info.json   → test5 PanoramaEvaluator（匹配统计）
  ├─ H_list.npy         → test5 PanoramaEvaluator（重投影误差）
  └─ inliers_list.pkl   → test5 PanoramaEvaluator（重投影误差）

analysis.py
  ├─ bridge_input()     → 时间单位转换 ms→s
  ├─ run_production_analysis()
  │   └─ PanoramaEvaluator
  │       ├─ calc_match_metrics()        → 总匹配对数、RANSAC内点数、内点率
  │       ├─ calc_reprojection_error()   → RMSE、中位数、P95
  │       ├─ calc_panorama_ssim()        → 全景SSIM、min、std  ★不依赖H矩阵
  │       ├─ calc_canvas_metrics()       → 有效画布占比、空白占比
  │       ├─ calc_sharpness_metrics()    → 清晰度保持率
  │       └─ get_composite_scores()      → 6维评分 + 综合得分
  ├─ build_eval_result()   → eval_result.json（6项摘要）
  ├─ build_full_metrics()  → full_metrics.json（分组详情）
  └─ generate_charts()     → PNG 仪表盘 / 雷达图 / 明细表

前端
  └─ GET /api/analysis/{task_id}/eval_result.json
      → 6 项指标表格 + 雷达图
```
