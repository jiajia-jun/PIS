# PIS-web 数据交互规范

本文档描述算法组（`stitch.py`）与数据分析组（`analysis.py`）之间的数据传递关系。

---

## 1. 整体流程

```
用户上传图片
    │
    ▼
┌────────── Go 后端 ────────────────┐
│  存入 store/uploads/{task_id}/input/  │
│  创建 store/uploads/{task_id}/result/ │
│  创建 store/analysis/{task_id}/       │
└────────────────────────────────────┘
    │
    ▼
┌─────── 第一步：算法组 stitch.py ────────┐
│  读 input/*.jpg                         │
│  执行拼接算法                            │
│  写 result/result.jpg                   │
│  写 result/meta.json                    │
└──────────────────────────────────────┘
    │ 拼接成功
    ▼
┌────── 第二步：数据分析组 analysis.py ─────┐
│  读 input/*.jpg                         │
│  读 result/result.jpg                   │
│  独立生成分析数据                        │
│  写 store/analysis/{task_id}/*.png      │
│  写 store/analysis/{task_id}/eval_result.json │
└──────────────────────────────────────┘
```

---

## 2. 目录全景

```
store/uploads/{task_id}/
├── input/                       # 算法组【读】| 分析组【读】
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── ...
└── result/                      # 算法组【写】| 分析组【读】
    ├── result.jpg               # 拼接全景图
    └── meta.json                # Go 后端专用，分析组不读

store/analysis/{task_id}/        # 分析组【写】
├── cost_trend.png
├── keypoints_chart.png
├── eval_result.json              # ← 新增：评估数据表格
└── ...
```

---

## 3. 算法组产出 → 分析组消费

| 文件 | 算法组 | 分析组 | 说明 |
|---|---|---|---|
| `input/img_*.jpg` | 读取并拼接 | 读取并分析 | 原始图片，双方都读 |
| `result/result.jpg` | **产出** | 读取 | 拼接结果，核心交接物 |
| `result/meta.json` | **产出** | **不读** | Go 后端用于前端展示历史记录 |

> 目前分析组直接从 `input/` 和 `result.jpg` 独立提取分析数据。

## 3.1 分析组产出 → 前端消费

| 文件 | 分析组 | 前端 | 说明 |
|---|---|---|---|
| `*.png` | **产出** | `<img>` 渲染 | 分析图表，数量不固定 |
| `eval_result.json` | **产出** | `<table>` 渲染 | 评估数据表格，固定文件名，格式为 `{title, headers, rows}` |

---

## 4. 如果分析组需要算法组提供额外数据

当前分析组不读 `meta.json`，如果需要算法组提供更多数据（如特征点坐标、匹配对数量等），算法组可以在 `result/` 下额外输出一个文件，例如 `stitch_detail.json`，分析组读取即可。

> ⚠️ 如需此文件，两组自行协商 JSON 结构，告知后端组记录到本文档中。

---

## 5. 数据交换总结

```
算法组 stitch.py                    分析组 analysis.py
─────────────────                  ───────────────────
读: input/*.jpg                   读: input/*.jpg
                                  读: result/result.jpg  ← 算法组产出
                                  
写: result/result.jpg  ──────────→ 消费
写: result/meta.json  ──────────→ Go 后端消费（非分析组）
                                 写: store/analysis/{task_id}/*.png
                                 写: store/analysis/{task_id}/eval_result.json  ← 新增
                                 	↑
                                 	└── 前端消费：图表区渲染 PNG，表格区渲染 JSON
```
