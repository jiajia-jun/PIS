# PIS-web 数据交互规范

本文档描述算法组（`stitch.py`）与数据分析组（`eval_pipeline.py`）之间的数据传递关系。

---

## 1. 整体流程

```
用户上传图片
    │
    ▼
┌────────── Go 后端 ──────────────────────────┐
│  存入 store/uploads/{task_id}/input/          │
│  创建 store/uploads/{task_id}/result/ (空目录) │
│  创建 store/analysis/{task_id}/      (空目录) │
└──────────────────────────────────────────────┘
    │
    ▼
┌─────── 第一步：算法组 stitch.py ───────────────┐
│  读 input/*.jpg                                 │
│  执行 OpenCV Stitcher 拼接                       │
│  写 result/result.jpg                           │
│  写 result/meta.json                            │
│  写 result/result_info.json  (匹配统计，供分析组) │
│  写 result/H_list.npy         (单应矩阵列表)     │
│  写 result/inliers_list.pkl   (内点坐标列表)     │
└────────────────────────────────────────────────┘
    │ 拼接成功
    ▼
┌────── 第二步：分析组 eval_pipeline.py ───────────┐
│  读 input/*.jpg                                 │
│  读 result/result.jpg                           │
│  读 result/result_info.json                     │
│  读 result/H_list.npy                           │
│  读 result/inliers_list.pkl                     │
│  计算 6 项指标 + 综合得分                         │
│  写 store/analysis/{task_id}/*.png  (10 张图表)  │
│  写 store/analysis/{task_id}/eval_result.json   │
│  写 store/analysis/{task_id}/full_metrics.json  │
└────────────────────────────────────────────────┘
```

---

## 2. 目录全景

```
store/uploads/{task_id}/
├── input/                       # 算法组【读】| 分析组【读】
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── ...
└── result/                      # 算法组【写】| 分析组【读】
    ├── result.jpg               # 拼接全景图
    ├── meta.json                # Go 后端专用（分析组不读）
    ├── result_info.json         # 逐对匹配统计 {pairs, total_time}
    ├── H_list.npy               # 相邻对单应矩阵 (n,3,3)
    └── inliers_list.pkl         # 相邻对内点坐标 [(N,4), …]

store/analysis/{task_id}/        # 分析组【写】| 前端【读】
├── eval_result.json             # 精简摘要（6 项指标）
├── full_metrics.json            # 分组详细数据
├── metrics_dashboard.png        # 六指标仪表盘总览
├── quality_radar.png            # 综合雷达图
├── gauge_内点率.png
├── gauge_重投影RMSE.png
├── gauge_全景SSIM.png
├── gauge_总耗时.png
├── gauge_画布利用率.png
├── gauge_综合得分.png
└── metrics_table.png            # 指标明细数据表
```

## 3. 算法组产出 → 分析组消费

| 文件 | 算法组 | 分析组 | 说明 |
|---|---|---|---|
| `input/img_*.jpg` | 读取并拼接 | 读取并计算清晰度 | 原始图片 |
| `result/result.jpg` | **产出** | 读取并分析 | 全景图，核心交接物 |
| `result/meta.json` | **产出** | **不读** | Go 后端专用（status, keypoints, cost_ms） |
| `result/result_info.json` | **产出** | 读取 | 逐对匹配统计 `{pairs: [{total_matches, inlier_num}], total_time}` |
| `result/H_list.npy` | **产出** | 读取 | 相邻图对单应矩阵，重投影误差计算用 |
| `result/inliers_list.pkl` | **产出** | 读取 | 相邻图对内点坐标，重投影误差计算用 |

---

## 4. 分析组产出 → 前端消费

| 文件 | 分析组 | 前端 | 说明 |
|---|---|---|---|
| `eval_result.json` | **产出** | `<table>` 摘要区 | 6 项指标精简摘要，固定文件名 |
| `full_metrics.json` | **产出** | `<el-tabs>` 详细面板 | 按 6 个分组展示全部字段 |
| `quality_radar.png` | **产出** | `<img>` 雷达图 | 六维综合雷达图 |
| `metrics_dashboard.png` | **产出** | `<img>` 图表网格 | 六指标仪表盘 |
| `gauge_*.png` ×6 | **产出** | `<img>` 图表网格 | 各项指标独立大图 |
| `metrics_table.png` | **产出** | `<img>` 图表网格 | 指标明细数据表 |

> PNG 图表数量固定 10 张，通过 `GET /api/analysis/{task_id}/{filename}` 访问。

---

## 5. 关键转换说明

### 5.1 时间单位转换

`stitch.py` 产出的 `result_info.json` 中 `total_time` 单位为**毫秒**。

`eval_pipeline.py` 的 `bridge_input()` 会将 `total_time` **从毫秒转换为秒**（÷1000），并备份原始文件为 `result_info_full.json`。分析组后续读取的是秒级值。

### 5.2 容错

- 拼接失败 → 不调用分析脚本
- 分析脚本失败 → 拼接依然标记为 `completed`，仅后端日志 `logs/task.log` 记录 warning
- 分析图表不存在 → 前端对应位置留空

## 6. 数据交换总结

```
stitch.py (算法组)                    eval_pipeline.py (分析组)
─────────────────                    ───────────────────────
读: input/*.jpg                      读: input/*.jpg
                                     读: result/result.jpg    ← 算法组产出
                                     读: result/result_info.json
                                     读: result/H_list.npy
                                     读: result/inliers_list.pkl

写: result/result.jpg  ────────────→ 消费
写: result/meta.json   ────────────→ Go 后端消费（非分析组）
写: result/result_info.json  ──────→ 消费
写: result/H_list.npy        ──────→ 消费
写: result/inliers_list.pkl  ──────→ 消费

                                     写: store/analysis/{task_id}/*.png
                                     写: store/analysis/{task_id}/eval_result.json
                                     写: store/analysis/{task_id}/full_metrics.json
                                        ↑
                                        └── 前端消费：图表区渲染 PNG，摘要区 + 详细面板渲染 JSON
```
